#+title: Remove npm/pnpm dependencies, swap Tailwind for the standalone v4 CLI
#+author: Darwin D. Wu
#+status: Draft
#+date: <2026-06-08 Mon>

Replace the npm-installed =@tailwindcss/cli= with the Tailwind v4 standalone
binary delivered through nix, so the repo no longer needs Node, pnpm, or a
=package.json= to build CSS.

* Background

The only consumer of npm in this repo is Tailwind's CLI. Hugo templates ship no
JavaScript built from =node_modules=, and CI workflows do not install npm
packages. =assets/css/app.css= uses Tailwind v4 syntax (=@import "tailwindcss"=,
=@plugin=, =@theme=, =@source inline(...)=), so the nixpkgs-provided Tailwind v3
binary cannot replace it. Tailwind v4 ships a single static binary on GitHub
Releases that bundles the typography plugin, which is the only plugin we use.

* Decisions

- Pin to *Tailwind v4.1.10* to match the version that produced the committed
  =assets/css/main.css=. Bumping to v4.3.0 happens in a follow-up so the
  toolchain change stays auditable.
- Drop =prettier= and =prettier-plugin-tailwindcss= entirely. Nothing in the
  repo invokes prettier today (no script, no hook, no CI step). A formatter can
  return later via nix if wanted.
- Keep using =Makefile= as the task entry point. Rename targets to =css=,
  =css-watch=, =build=, =serve=.

* Out of scope

  attributes across 16 templates plus the =prose= typography plugin on post

  the generated =main.css= stays attributable.
